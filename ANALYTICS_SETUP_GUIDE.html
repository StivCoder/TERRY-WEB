📊 Analytics Dashboard Setup Guide

Complete guide to setting up the advanced analytics system for Terry's Tender Wear

📖 What's New

Your store now has a powerful analytics system that tracks:

  • User Sessions: How many unique visitors, session duration, device type
  • Page Analytics: Most visited pages, time spent on each page
  • Product Analytics: Product views, clicks, add-to-cart interactions, conversion rates
  • Sales Analytics: Revenue trends, total orders, average order value
  • Engagement Metrics: Bounce rate, pages per session, return visitors, conversion rate

🚀 Quick Start

For Local Storage Mode (No Setup Required)

Analytics works immediately! Data is stored locally in your browser.

  1. Visit any page on your store (index.html, products.html, etc.)
  2. Log into your admin panel
  3. Click 📈 Analytics in the sidebar
  4. View your analytics dashboard
ℹ️ Note: Local storage analytics are only visible in the same browser. For persistent analytics across devices, configure Supabase below.

🗄️ Supabase Setup (Optional but Recommended)

For persistent analytics across all devices and browsers, configure Supabase:

Step 1: Get Your Supabase Credentials

  1. Open supabase.com
  2. Login and select your project
  3. Go to Settings → API in the left sidebar
  4. Copy your Project URL and Anon Public Key

Step 2: Update supabase-config.js

Open supabase-config.js and paste your credentials:

var SUPABASE_URL = "https://YOUR_URL.supabase.co";
var SUPABASE_ANON_KEY = "YOUR_ANON_KEY_HERE";

Step 3: Create Analytics Tables

  1. In Supabase, click SQL Editor in the left sidebar
  2. Click New Query
  3. Copy the SQL from supabase-config.js comments (the SQL for analytics tables)
  4. Paste it into Supabase and click Run
✅ Done! Your analytics database is now set up. Data collection starts automatically.

🎯 How Analytics Work

What Gets Tracked Automatically

  • Every page visit
  • Time spent on each page
  • Device type (Mobile/Tablet/Desktop)
  • Product views and add-to-cart actions
  • Every sale/order

Session Tracking

Each visitor gets a unique session ID stored in their browser. Sessions last 30 minutes of inactivity.

Privacy & Security

  • No personal information is tracked
  • Data is admin-only with Supabase Row Level Security
  • Your Supabase database is encrypted

💡 Tips for Success

  • Check weekly: Monitor trends and adjust strategy
  • Study top products: Allocate more inventory to best sellers
  • Improve low performers: Better descriptions for products with low conversion
  • Optimize mobile: Most traffic comes from mobile devices
  • Track seasons: Note traffic patterns (holidays, sales, etc.)

❓ FAQ

Do I need Supabase to use analytics?

No! Analytics work with just local storage. But with Supabase, data persists across devices.

How accurate is the tracking?

The system tracks JavaScript events. If a user has JavaScript disabled or ad blockers on, events won't be tracked. This is standard for all web analytics.

Can I export the data?

With Supabase, you can export data directly from the database. Contact support for help.

How long is data stored?

Analytics data is stored indefinitely. You can delete old records manually if needed.

🆘 Troubleshooting

Analytics dashboard is empty

Solution:

  • Wait a few minutes for data to collect (users need to visit your store)
  • Check that Supabase is properly configured
  • Verify the SQL queries were run in Supabase
  • Open browser Developer Tools (F12) and check Console for errors

I see data in analytics, but numbers seem low

Solution:

  • Analytics only track real user sessions (not bot traffic or pre-existing caches)
  • Make sure you've visited your store since setting up analytics
  • Refresh the dashboard to reload latest data

Supabase is giving an error

Solution:

  • Double-check your Project URL and Anon Key in supabase-config.js
  • Make sure the SQL queries are executed successfully in Supabase
  • Verify RLS (Row Level Security) policies exist on analytics tables

📞 Need Help?

If you encounter issues:

  1. Open your browser's Developer Tools (F12 or right-click → Inspect)
  2. Go to the Console tab
  3. Look for any red error messages
  4. Share the error message when seeking help
