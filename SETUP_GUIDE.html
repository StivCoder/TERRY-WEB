1

Create Your Supabase Project

Go to supabase.com and sign up for free. Then create a new project.

  • Click "New Project"
  • Name it: terry-tender-wear
  • Set a strong database password (save it!)
  • Choose region: Europe West (closest to Kenya)
  • Click "Create new project" — wait ~2 minutes
✅ Free plan includes 500MB storage and 2GB bandwidth — more than enough for your store.
2

Get Your API Keys

In your Supabase dashboard, click Settings ⚙️ → API in the left sidebar.

  • Copy Project URL → looks like https://abcxyz.supabase.co
  • Copy anon / public key → long string starting with eyJ…
⚠️ Never copy the service_role key into your website files — it has full database access with no restrictions.
3

Paste Your Keys into supabase-config.js

Open supabase-config.js in any text editor and replace the two placeholder values:

supabase-config.js
// ✏️ EDIT THESE TWO LINES ONLY:
var SUPABASE_URL      = "https://YOUR_PROJECT_ID.supabase.co";
var SUPABASE_ANON_KEY = "YOUR_ANON_PUBLIC_KEY_HERE";

// Replace with your real values, for example:
var SUPABASE_URL      = "https://abcdefghijk.supabase.co";
var SUPABASE_ANON_KEY = "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...";

Save the file and re-upload it to GitHub.

4

Create Database Tables

In Supabase, go to SQL Editor → New Query. Paste and run this SQL:

Run in Supabase SQL Editor
-- Products table
CREATE TABLE IF NOT EXISTS products (
  id          BIGSERIAL    PRIMARY KEY,
  name        TEXT         NOT NULL,
  price       INTEGER      NOT NULL CHECK (price > 0),
  category    TEXT         NOT NULL DEFAULT '',
  description TEXT         NOT NULL DEFAULT '',
  image_url   TEXT         NOT NULL DEFAULT '',
  stock       INTEGER      NOT NULL DEFAULT 0,
  is_new      BOOLEAN      NOT NULL DEFAULT false,
  is_featured BOOLEAN      NOT NULL DEFAULT false,
  created_at  TIMESTAMPTZ  NOT NULL DEFAULT NOW()
);

-- Orders table
CREATE TABLE IF NOT EXISTS orders (
  id             BIGSERIAL    PRIMARY KEY,
  customer_name  TEXT         NOT NULL DEFAULT '',
  customer_phone TEXT         NOT NULL DEFAULT '',
  items          JSONB        NOT NULL DEFAULT '[]',
  total          INTEGER      NOT NULL DEFAULT 0,
  status         TEXT         NOT NULL DEFAULT 'pending',
  notes          TEXT         NOT NULL DEFAULT '',
  created_at     TIMESTAMPTZ  NOT NULL DEFAULT NOW()
);
✅ Click the green Run button. You should see "Success. No rows returned."
5

Create the Image Storage Bucket

This is where product images get stored. This step is required for images to appear on the frontend.

⚠️ Without this step, images upload but show broken on the shop page.

Option A — Via Dashboard (easiest):

  • Go to Storage in the left sidebar
  • Click "New bucket"
  • Name it exactly: product-images
  • Toggle "Public bucket" → ON ✅ (critical!)
  • Click Save

Option B — Via SQL Editor:

-- Create public storage bucket for product images
INSERT INTO storage.buckets (id, name, public)
VALUES ('product-images', 'product-images', true)
ON CONFLICT (id) DO UPDATE SET public = true;
🔑 The bucket MUST be set to Public. If it's private, images upload successfully but return a 403 error when the browser tries to display them.
6

Set Up Security Policies (RLS)

These rules control who can read and write data. Run all of this in the SQL Editor:

Run in Supabase SQL Editor
-- Enable Row Level Security
ALTER TABLE products ENABLE ROW LEVEL SECURITY;
ALTER TABLE orders   ENABLE ROW LEVEL SECURITY;

-- Drop old policies if any exist
DROP POLICY IF EXISTS "Public read products"  ON products;
DROP POLICY IF EXISTS "Admin insert products" ON products;
DROP POLICY IF EXISTS "Admin update products" ON products;
DROP POLICY IF EXISTS "Admin delete products" ON products;
DROP POLICY IF EXISTS "Public insert orders"  ON orders;
DROP POLICY IF EXISTS "Admin read orders"    ON orders;
DROP POLICY IF EXISTS "Admin update orders"  ON orders;

-- Anyone can VIEW products (public storefront)
CREATE POLICY "Public read products"
  ON products FOR SELECT
  TO anon, authenticated
  USING (true);

-- Logged-in admin can ADD products
CREATE POLICY "Admin insert products"
  ON products FOR INSERT
  TO authenticated
  WITH CHECK (auth.uid() IS NOT NULL);

-- Logged-in admin can EDIT products
CREATE POLICY "Admin update products"
  ON products FOR UPDATE
  TO authenticated
  USING (auth.uid() IS NOT NULL)
  WITH CHECK (auth.uid() IS NOT NULL);

-- Logged-in admin can DELETE products
CREATE POLICY "Admin delete products"
  ON products FOR DELETE
  TO authenticated
  USING (auth.uid() IS NOT NULL);

-- Anyone can LOG an order (WhatsApp checkout)
CREATE POLICY "Public insert orders"
  ON orders FOR INSERT
  TO anon, authenticated
  WITH CHECK (true);

-- Admin can VIEW all orders
CREATE POLICY "Admin read orders"
  ON orders FOR SELECT
  TO authenticated
  USING (auth.uid() IS NOT NULL);

-- Admin can UPDATE order status
CREATE POLICY "Admin update orders"
  ON orders FOR UPDATE
  TO authenticated
  USING (auth.uid() IS NOT NULL)
  WITH CHECK (auth.uid() IS NOT NULL);

-- Storage: anyone can VIEW images
CREATE POLICY "Public read images"
  ON storage.objects FOR SELECT
  TO anon, authenticated
  USING (bucket_id = 'product-images');

-- Admin can UPLOAD images
CREATE POLICY "Admin upload images"
  ON storage.objects FOR INSERT
  TO authenticated
  WITH CHECK (bucket_id = 'product-images' AND auth.uid() IS NOT NULL);

-- Admin can DELETE images
CREATE POLICY "Admin delete images"
  ON storage.objects FOR DELETE
  TO authenticated
  USING (bucket_id = 'product-images' AND auth.uid() IS NOT NULL);
7

Create Your Admin Account

In Supabase, go to Authentication → Users → Add user → Create new user.

  • Enter your email address (e.g. admin@terrys-tenderwear.co.ke)
  • Set a strong password
  • Click "Create user"

Use this exact email and password to log into admin-login.html.

💡 You can also tick "Auto Confirm User" so you don't need to verify the email.
8

Test Everything Works

Open your live site and check each of these:

Connection Checklist

Open admin-login.html — badge shows "Supabase Connected" (green)
Login with your Supabase email & password — goes to dashboard
Add a product with an image upload — green success message appears
Open products.html — the new product appears with its image
Add product to cart and complete WhatsApp checkout — order appears in admin Orders tab
Delete a product from admin — disappears from shop immediately

🔧 Troubleshooting

?

Images upload but don't appear on the shop

  • Your product-images bucket is private — go to Storage → bucket → Settings → make it Public
  • The "Public read images" storage policy was not created — re-run Step 6 SQL
  • Open browser DevTools (F12) → Network tab → find the image request → check if it returns 403
?

Login says "Invalid credentials"

  • The email/password must match the user created in Supabase → Authentication → Users
  • Check for typos in both email and password
  • In Supabase Authentication settings, make sure Email confirmations are disabled or the user is already confirmed
  • Go to Authentication → Users → find your user → click the 3 dots → "Send password recovery" to reset
?

Products not showing on the shop

  • Open browser DevTools (F12) → Console tab — look for red error messages
  • Check that supabase-config.js has your real URL and key (not the placeholder text)
  • Re-run the "Public read products" policy from Step 6
  • Check Supabase → Table Editor → products — confirm rows exist
?

Admin can't add/edit/delete products

  • You must be logged in — the session token is required for write operations
  • Check that the "Admin insert/update/delete products" policies from Step 6 were created
  • In Supabase → Authentication → Policies → products table — verify all 4 policies are listed
