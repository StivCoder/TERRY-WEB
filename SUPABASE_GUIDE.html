📘 Setup Guide

Connect Supabase to Your Store

This guide will walk you through creating a free Supabase database and linking it to Terry's Tender Wear so your products and orders sync across all devices.

✅ Good news: The store works perfectly right now using Local Storage — no setup needed. Follow this guide only when you're ready for a real database.
1

Create a free Supabase account

Go to supabase.com and click "Start for free". Sign up with GitHub, Google, or your email address.

Once signed in, click "New project". Give it the name terry-tender-wear, set a strong database password (save it!), and choose a region close to Kenya — Europe West works well.

Click "Create new project" and wait about 2 minutes for Supabase to finish setup.

⏱ The free plan is completely free forever with up to 500 MB storage — more than enough for a product catalogue.
2

Get your API keys

In your Supabase project dashboard:

1. Click the ⚙️ Settings icon in the left sidebar.
2. Click "API" under Configuration.
3. Copy your Project URL (looks like https://abcxyz.supabase.co).
4. Copy the anon / public key (long string starting with eyJ…).

⚠️ Never share your "service_role" key — it has full database access. Only use the anon/public key in your website files.
3

Paste your keys into supabase-config.js

Open the file 📄 supabase-config.js in any text editor and replace the two placeholder values:

// BEFORE (placeholders): const SUPABASE_URL = "https://YOUR_PROJECT_ID.supabase.co"; const SUPABASE_ANON_KEY = "YOUR_ANON_PUBLIC_KEY_HERE"; // AFTER (your real values): const SUPABASE_URL = "https://abcxyzabc.supabase.co"; const SUPABASE_ANON_KEY = "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...";

Save the file. The store will automatically switch from Local Storage to Supabase.

4

Create the database tables

In Supabase, click "SQL Editor" in the left sidebar, then click "New query". Paste the SQL below and click "Run":

-- Products table CREATE TABLE products ( id BIGSERIAL PRIMARY KEY, name TEXT NOT NULL, price INTEGER NOT NULL, category TEXT NOT NULL, description TEXT DEFAULT '', image_url TEXT DEFAULT '', stock INTEGER DEFAULT 0, is_new BOOLEAN DEFAULT false, is_featured BOOLEAN DEFAULT false, created_at TIMESTAMPTZ DEFAULT NOW() ); -- Orders table (WhatsApp orders are logged here) CREATE TABLE orders ( id BIGSERIAL PRIMARY KEY, customer_name TEXT, customer_phone TEXT, items JSONB, total INTEGER, status TEXT DEFAULT 'pending', created_at TIMESTAMPTZ DEFAULT NOW() ); -- Row Level Security ALTER TABLE products ENABLE ROW LEVEL SECURITY; ALTER TABLE orders ENABLE ROW LEVEL SECURITY; -- Anyone can read products (public storefront) CREATE POLICY "Public read products" ON products FOR SELECT USING (true); -- Anyone can insert orders (customers placing orders) CREATE POLICY "Public insert orders" ON orders FOR INSERT WITH CHECK (true); -- Authenticated users (admin) have full access CREATE POLICY "Admin full products" ON products FOR ALL USING (auth.role() = 'authenticated'); CREATE POLICY "Admin full orders" ON orders FOR ALL USING (auth.role() = 'authenticated');

6

Create your admin account

In Supabase, go to Authentication → Users and click "Add user". Enter your email address and a strong password.

This is the email and password you'll use on the Admin Login page. When Supabase is connected, the login uses real Supabase Auth — much more secure than the Local Storage fallback.

💡 You can also enable Google / GitHub social login for the admin under Authentication → Providers in Supabase — completely optional.
7

Deploy your website (optional)

Your website is a collection of plain HTML/CSS/JS files — no server required. You can host it free on any of these platforms:

• GitHub Pages — Free, push your files to a GitHub repo and enable Pages.
• Netlify — Drag and drop your folder. Live in 30 seconds. Free plan available.
• Vercel — Same as Netlify. Free plan available.

⚠️ When deploying publicly, make sure your Supabase Row Level Security (RLS) policies are set correctly (Step 4 above already does this). Your anon key is safe to include in public files — RLS controls what it can access.

Troubleshooting

?

Products aren't loading from Supabase

• Double-check your SUPABASE_URL and SUPABASE_ANON_KEY in supabase-config.js — even one wrong character breaks it.
• Make sure you ran the SQL in Step 4 to create the products table.
• Check the browser Console (F12 → Console) for error messages.
• Confirm Row Level Security is enabled and the Public read policy exists.

?

Admin login says "Invalid credentials"

• When Supabase is configured, you must use the email+password from your Supabase Auth user (Step 6).
• The fallback password terry2024 only works in Local Storage mode (no Supabase configured).
• Reset your password in Supabase → Authentication → Users.

?

How do I change the WhatsApp number?

Open 📄 supabase-config.js and change the line:
const WHATSAPP_NUMBER = "254118053640";
Replace 254118053640 with your number in international format — no + sign, no spaces. For example: 254700123456
